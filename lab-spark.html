Theory: Apache Spark Fundamentals

What is Apache Spark?

Apache Spark is a unified analytics engine for large-scale data processing. Unlike Hadoop MapReduce which writes to disk between stages, Spark keeps data in memory, making it 10-100x faster for iterative algorithms.

Resilient Distributed Datasets (RDDs)

RDDs are immutable, distributed collections of objects that can be processed in parallel. They provide fault tolerance through lineage tracking.

Note: While RDDs are the foundation, DataFrames are now preferred for most use cases due to better optimization.

Lazy Evaluation & DAGs

Spark doesn't compute results immediately. Instead, it builds a Directed Acyclic Graph (DAG) of transformations. Only when you call an action (like collect(), write(), or show()) does Spark execute the computation.

Spark Ecosystem

  • Spark SQL: SQL queries on distributed data
  • MLlib: Machine learning library
  • Spark Streaming: Real-time data processing
  • GraphX: Graph processing

Exercise 2: MapReduce Concepts

Learn the map-reduce pattern and apply it to MegaStore data.

Part A: Classic Word Count

First, implement word count using RDD operations:

# Word Count MapReduce Example
from pyspark.sql import SparkSession

spark = SparkSession.builder.appName("WordCount").getOrCreate()

# Sample text
text = """spark is fast spark is fun spark is powerful
hadoop is slow hadoop is old hadoop is legacy"""

# Create RDD
rdd = spark.sparkContext.parallelize([text])

# Map: split into words
words = rdd.flatMap(lambda line: line.split())

# Map: create (word, 1) pairs
pairs = words.map(lambda word: (word, 1))

# Reduce: sum by key
word_counts = pairs.reduceByKey(lambda a, b: a + b)

# Sort by count descending
result = word_counts.sortByKey().collect()
for word, count in result:
    print(f"{word}: {count}")

Part B: MegaStore Order Analysis

Apply MapReduce to count orders by category:

# Load data
orders_df = spark.read.parquet("/home/jovyan/data/orders.parquet")
order_items_df = spark.read.parquet("/home/jovyan/data/order_items.parquet")
products_df = spark.read.csv("/home/jovyan/data/products.csv", header=True)

# Convert to RDD and apply MapReduce
# Map: emit (category, 1) for each order
orders_rdd = order_items_df.rdd \
    .map(lambda row: (row.category, 1))

# Reduce: sum by category
category_counts = orders_rdd.reduceByKey(lambda a, b: a + b)

# Collect and display
for category, count in category_counts.collect():
    print(f"{category}: {count} items")

# Query 1: Top 10 customers by revenue
spark.sql("""
    SELECT
        c.customer_name,
        COUNT(o.order_id) as num_orders,
        SUM(oi.quantity * oi.unit_price) as total_revenue
    FROM orders o
    JOIN customers c ON o.customer_id = c.customer_id
    JOIN order_items oi ON o.order_id = oi.order_id
    GROUP BY c.customer_name
    ORDER BY total_revenue DESC
    LIMIT 10
""").show()

# Query 2: Revenue by product category
spark.sql("""
    SELECT
        p.category,
        COUNT(DISTINCT o.order_id) as num_orders,
        SUM(oi.quantity) as total_items,
        ROUND(SUM(oi.quantity * oi.unit_price), 2) as total_revenue
    FROM order_items oi
    JOIN products p ON oi.product_id = p.product_id
    JOIN orders o ON oi.order_id = o.order_id
    GROUP BY p.category
    ORDER BY total_revenue DESC
""").show()

# Query 3: Orders per month
spark.sql("""
    SELECT
        DATE_TRUNC('month', o.order_date) as month,
        COUNT(*) as num_orders,
        AVG(o.order_total) as avg_order_value
    FROM orders o
    GROUP BY DATE_TRUNC('month', o.order_date)
    ORDER BY month DESC
""").show()

Exercise 4: Complete ETL Pipeline

Build an end-to-end pipeline: Load → Clean → Transform → Aggregate → Save.

from pyspark.sql.functions import *
from pyspark.sql.types import *

spark = SparkSession.builder.appName("ETLPipeline").getOrCreate()

# 1. LOAD
print("Step 1: Loading raw data...")
clickstream_df = spark.read.parquet("/home/jovyan/data/clickstream.parquet")
products_df = spark.read.csv("/home/jovyan/data/products.csv", header=True)

# 2. CLEAN
print("Step 2: Cleaning data...")
# Remove bots (identified by bot_id field)
clean_df = clickstream_df.filter(~col("bot_flag"))
# Remove nulls in critical fields
clean_df = clean_df.dropna(subset=["user_id", "session_id"])
# Remove duplicates
clean_df = clean_df.dropDuplicates(["user_id", "session_id", "timestamp"])

print(f"After cleaning: {clean_df.count()} records")

# 3. TRANSFORM
print("Step 3: Transforming data...")
# Calculate session duration
transformed_df = clean_df \
    .groupBy("user_id", "session_id") \
    .agg(
        min("timestamp").alias("session_start"),
        max("timestamp").alias("session_end"),
        count("*").alias("num_events"),
        countDistinct("page").alias("num_pages")
    ) \
    .withColumn(
        "session_duration_seconds",
        (unix_timestamp(col("session_end")) - unix_timestamp(col("session_start")))
    )

# 4. AGGREGATE
print("Step 4: Aggregating metrics...")
metrics_df = transformed_df \
    .withColumn("avg_session_duration", avg("session_duration_seconds").over()) \
    .groupBy(window("session_start", "1 day").alias("day")) \
    .agg(
        count("session_id").alias("num_sessions"),
        avg("session_duration_seconds").alias("avg_duration"),
        avg("num_events").alias("avg_events_per_session"),
        percentile_approx("session_duration_seconds", 0.5).alias("median_duration")
    )

# 5. OUTPUT
print("Step 5: Saving results...")
metrics_df.write.mode("overwrite").parquet("/home/jovyan/data/output/etl_metrics")
print("ETL pipeline completed successfully!")

Self-Check Questions

Test your understanding of Spark fundamentals. Click to reveal answers:

What is the difference between a transformation and an action in Spark?

Transformations (map, filter, join) build the execution plan (DAG) but don't compute results. Actions (collect, count, write) trigger actual execution of the plan. Spark uses lazy evaluation to optimize the entire pipeline before running.

Why does Spark use lazy evaluation?

Lazy evaluation allows Spark's Catalyst optimizer to see the entire computation plan before execution. This enables optimizations like predicate pushdown, constant folding, and stage coalescence. The result is faster execution than eager evaluation would allow.

What is a DataFrame in Spark and how does it differ from a Pandas DataFrame?

Spark DataFrames are distributed, immutable collections with schema enforcement and SQL support. Pandas DataFrames are in-memory, mutable, and single-machine. Spark DataFrames scale to petabytes, while Pandas is limited to machine memory.

What is partitioning and why is it important for performance?

Partitioning divides data into chunks distributed across the cluster. Proper partitioning enables parallelism, reduces network shuffle, and improves query performance. Partitioning by date/region allows Spark to prune irrelevant data during queries.

When would you use Spark SQL vs. DataFrame API?

Use Spark SQL for complex analytical queries and when collaborating with SQL-familiar analysts. Use DataFrame API for procedural logic, machine learning pipelines, and when you need programmatic control. Both compile to the same optimized execution plan.

Task 3: User Defined Functions (UDFs)

from pyspark.sql.functions import udf
from pyspark.sql.types import DoubleType, StringType

# Define a Python function
def calculate_discount(order_total):
    if order_total > 1000:
        return 0.2  # 20% discount
    elif order_total > 500:
        return 0.1  # 10% discount
    else:
        return 0.0  # No discount

# Convert to UDF
discount_udf = udf(calculate_discount, DoubleType())

# Apply to DataFrame
orders_df = spark.read.parquet("/home/jovyan/data/orders.parquet")
discounted = orders_df.withColumn(
    "discount",
    discount_udf(col("order_total"))
).withColumn(
    "final_price",
    col("order_total") * (1 - col("discount"))
)

discounted.select("order_id", "order_total", "discount", "final_price").show()

# String UDF example
def categorize_order(total):
    if total < 50:
        return "small"
    elif total < 200:
        return "medium"
    else:
        return "large"

category_udf = udf(categorize_order, StringType())
categorized = orders_df.withColumn("order_category", category_udf(col("order_total")))
categorized.select("order_id", "order_total", "order_category").show()

Summary & Troubleshooting

Key Takeaways

Distributed Processing

Spark distributes data and computation across clusters, enabling processing of datasets larger than single-machine memory

Lazy Evaluation

Spark builds execution plans (DAGs) before running; this enables optimization and only materializes data when needed

DataFrame API

DataFrames are more efficient than RDDs for most operations; they enable SQL queries and Catalyst optimization

ETL Pipelines

Structure data processing as Extract-Transform-Load pipelines for maintainability, reusability, and fault tolerance

Performance Matters

Choose the right file format (Parquet), partition wisely, cache intermediate results, and monitor with Spark UI

Common Issues & Solutions

Out of Memory (OOM) Errors

Solution: Increase executor memory with --executor-memory flag, use partitioning, or cache selectively

Slow Performance

Solution: Check Spark UI (port 4040) for bottlenecks, use Parquet instead of CSV, enable caching for reused DataFrames

Data Skew

Solution: Repartition data before joins/groupBys, use salting techniques for highly skewed keys

Connection Issues

Solution: Verify Docker container is running, check port forwarding (8888, 4040), inspect logs with docker logs <container_id>
